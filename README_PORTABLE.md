# Перенос проекта на флешке и запуск на новом ПК

## Что нужно

- Флешка
- Новый ПК с Windows
- Доступ к установке Python (если Python еще не установлен)

## 1. Копирование проекта на флешку

На текущем ПК:

1. Закройте Excel-файлы отчета, если они открыты.
2. Скопируйте всю папку проекта `monthly_report` на флешку.

## 2. Копирование проекта с флешки на новый ПК

На новом ПК:

1. Скопируйте папку `monthly_report` с флешки в удобное место, например:
   `C:\Users\<ваш_пользователь>\Desktop\monthly_report`
2. Откройте PowerShell в папке проекта.

## 3. Подготовка к работе одной командой

Выполните в корне проекта:

```powershell
powershell -ExecutionPolicy Bypass -File .\setup.ps1
```

Что произойдет автоматически:

- создастся виртуальное окружение `.venv`,
- обновится `pip`,
- установятся зависимости (`pandas`, `openpyxl`).

## 4. Запуск сверки

После подготовки можно запускать двойным кликом по файлу (без окна терминала):

`Сверка банка.vbs`

Альтернативно (может кратко мигнуть консоль):

`Сверка банка.bat`

Этот файл откроет графический интерфейс (`--gui`) для выбора:
- файла OC,
- файла 1С,
- пути сохранения отчета.

Или с явными файлами:

```powershell
.\.venv\Scripts\python.exe .\compare_payments.py --file1 work/OC.xlsx --file2 work/1C.xlsx --out reports/payments_comparison.xlsx
```

Или запустите графический режим:

```powershell
.\.venv\Scripts\python.exe .\compare_payments.py --gui
```

Далее выберите 2 входных файла и путь сохранения отчета, затем нажмите `Сформировать отчет`.

Если при двойном клике появится:
`[ERROR] .venv is not found.`
значит шаг 3 еще не выполнен или выполнен не в той папке.
Повторите:

```powershell
powershell -ExecutionPolicy Bypass -File .\setup.ps1
```

## 5. Если Python не установлен

1. Установите Python 3.10+ с официального установщика.
2. Повторите команду подготовки:

```powershell
powershell -ExecutionPolicy Bypass -File .\setup.ps1
```
